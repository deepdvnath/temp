Node2Vec: The Complete Process

From Graph to Vector Embeddings in 4 Simple Steps

1
Start with a Graph

You have a graph with nodes (vertices) and edges (connections).

Examples:

  • Social network (people and friendships)
  • Web pages and links
  • Molecules and chemical bonds
  • Cities and roads
Goal: Convert each node into a numerical vector that captures its position and role in the network.
2
Generate Random Walks (Using p and q)

Start from each node and take random walks through the graph.

Key: Use p and q parameters to bias the walk:

  • p controls returning to previous node
  • q controls exploring close vs. far

To Training Pairs:

(A, B), (A, E), (B, A), (B, E), (B, G)
(E, B), (E, G), (E, F), (G, B), (G, E)
...

Treat the random walks like sentences and nodes like words.

For each node in a walk:

  • Create pairs with nearby nodes (context window)
  • These pairs become training data
If two nodes appear together often in walks, they should have similar embeddings.

This is exactly like Word2Vec! Words that appear in similar contexts get similar vectors.

4
Train Neural Network to Learn Embeddings

Neural Network Training:

  • Input: A node (like node A)
  • Output: Predict its context nodes (B, D, E...)
  • Hidden layer weights = Node embeddings!
# Simplified training process
for (center_node, context_node) in training_pairs:
  prediction = model(center_node)
  loss = compute_loss(prediction, context_node)
  update_embeddings(loss)
After training, each node has a vector that captures its network position and relationships.
